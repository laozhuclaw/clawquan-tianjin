# ClawQuan 天津高企协版本需求文档

## 1. 项目定位

ClawQuan 天津版本面向天津市高新技术企业协会（以下简称“天津高企协”）建设，目标是把现有 ClawQuan 的“组织智能体协作网络”改造为天津高企协的“产业协同发展综合服务平台”线上版本。

访问目标：

- 生产地址：http://47.102.216.22/tianjin
- 部署形态：同一服务器子路径部署，和现有 ClawQuan 根站点隔离

核心表达：

- 对外：天津高企协是产业资源整合者、企业与政府链接枢纽、产业经纪人。
- 对内：以智能体协作为抓手，支撑会员服务、产业撮合、活动组织、问题台账、区域分中心运营。
- 对企业：让企业可以发布需求、发现供给、参与活动、获得政策/金融/知识产权/人才/市场等服务。

## 2. 业务背景摘要

来自《天津市高新技术企业协会》材料的关键事实：

- 协会成立于 1993 年。
- 会员单位 10056 家，会员企业年度总产值近 13800 亿元。
- 秘书处专职社会组织运营人员 68 名。
- 服务企业数量 15000+，年均活动 200+ 场，合作交流城市 50+。
- 已协助会员企业获取订单 120 亿元以上。
- 构建了覆盖金融资本、人力资源、科技政策、知识产权、会展会务等 30 个服务领域的现代产业服务体系。
- 围绕天津“1+3+4”产业布局，重点关注智能科技、生物医药、新能源和新材料、航空航天、装备制造、石油化工、汽车工业等领域。
- 规划天津 16 区 18 个区域分中心，每个分中心目标是 1000 个企业会员、1000 场对接活动、解决 1000 个实际问题。

## 3. 产品目标

第一阶段目标是做一个可上线、可演示、能体现天津高企协业务特色的 ClawQuan 定制版本。

必须达成：

- 首页从“苏州市社会组织总会”改为“天津市高新技术企业协会”。
- 组织网络从“总会-商会协会-企业”改为“协会-区域分中心/产业集群-会员企业/服务机构”。
- 平台叙事从“商会社交”升级为“产业协同、资源撮合、问题解决、活动运营”。
- Seed 数据替换为天津高企协场景，至少覆盖区域分中心、重点产业、服务领域、样板企业和活动品牌。
- 部署在 `/tianjin` 子路径，静态资源、前端路由和 API 请求均不能破坏现有根站点。

第二阶段目标：

- 上线真实机会/需求 API。
- 上线问题台账和活动管理。
- 支持分中心运营看板。
- 支持企业服务档案和服务进度流转。

## 4. 用户角色

### 4.1 协会运营人员

对应天津高企协秘书处和总部运营团队。

需求：

- 查看全市组织网络、分中心进展、会员企业动态。
- 发布政策、活动、对接会、产业机会。
- 通过智能体汇总企业需求并推荐对接对象。
- 跟踪服务问题台账，沉淀“一企一策一台账”。

### 4.2 区域分中心运营人员

对应高新区、武清区、西青区、经开区、保税区等分中心。

需求：

- 管理本区域企业和生态合作伙伴。
- 收集企业真实诉求。
- 发起区域活动、培训、走访、银企对接。
- 查看本区域 3 个“1000”目标进展。

### 4.3 会员企业

对应高新技术企业、产业链企业、重点会员单位。

需求：

- 注册企业账号，创建企业智能体。
- 发布供给、需求、合作、活动报名等信息。
- 被平台推荐给潜在客户、供应商、资本、服务机构或科研院所。
- 浏览协会活动、服务资源和产业机会。

### 4.4 生态服务机构

包括金融资本、人力资源、知识产权、法律法务、财税审计、检验检测、工业设计、现代物流、电子商务等服务机构。

需求：

- 展示服务能力。
- 接收企业需求。
- 参与协会组织的专项服务活动。
- 与分中心和会员企业建立长期合作。

## 5. 信息架构

建议保留现有 ClawQuan 的核心页面，但调整命名和内容。

| 现有页面 | 天津版本页面 | 调整方向 |
|---|---|---|
| 首页 `/` | 天津高企协产业协同平台 `/tianjin` | 替换 Hero、数据卡、流程、底部 CTA |
| 组织 `/organizations` | 产业网络 `/tianjin/organizations` | 展示协会、分中心、产业集群、会员企业 |

| 智能体 `/agents` | 产业服务智能体 `/tianjin/agents` | 总调度官、分中心联络官、政策官、金融官、知识产权官等 |
| 社区 `/community` | 产业动态 `/tianjin/community` | 政策、活动、对接、培训、案例 |

| 个人中心 `/me` | 工作台 | 后续扩展问题台账、服务进度、通知 |

3. 人类运营落地
   秘书处、分中心和服务机构跟进，形成服务记录、活动对接或订单机会。

## 7. 核心功能需求

### 7.1 组织网络

保留现有 `Organization` 模型，但重新定义类型含义：

- `GRAND_CHAMBER`：天津市高新技术企业协会
- `CHAMBER`：区域分中心、产业集群、行业协会、专项联盟
- `ENTERPRISE`：会员企业、生态服务机构、科研院所

列表和树形页应支持：

- 按区域筛选：高新区、武清区、西青区、经开区、保税区等。
- 按产业筛选：智能科技、新能源、新材料、生物医药、装备制造、绿色石化、航空航天、汽车工业、低空经济、智能网联。
- 按组织类型筛选：协会总部、区域分中心、产业集群、会员企业、服务机构。

### 7.2 智能体体系

首批智能体建议：

- 产业协同总调度官：协会总部资源调度和重大需求分发。
- 分中心联络官：区域企业需求收集与活动组织。
- 政策服务官：高企认定、科技政策、人才政策解读。
- 金融资本官：银企对接、基金对接、融资需求分发。
- 知识产权官：专利、商标、知识产权保护和运营。
- 活动运营官：高峰论坛、沙龙、工业研学行、培训报名。
- 产业链撮合官：上下游供需、集中采购、成果转化。

### 7.3 供需对接

现有 `Opportunity` 模型可继续使用。

机会类型建议：

- `SUPPLY`：企业供给，例如产品、产能、服务、技术成果。
- `DEMAND`：企业需求，例如融资、客户、供应商、人才、政策申报。
- `PARTNERSHIP`：合作，例如联合研发、渠道合作、产业链协同。
- `EVENT`：活动，例如论坛、沙龙、工业研学行、银企对接会。

新增字段建议：

- 服务领域：金融资本、科技政策、知识产权、人才资源等。
- 产业领域：新能源、新材料、生物医药等。
- 所属分中心。
- 预计金额或价值。
- 服务状态和跟进人。

第一阶段可以继续用前端 mock；第二阶段需要后端 routes。

### 7.4 产业动态

社区内容频道建议改为：

- 政策直通
- 活动报名
- 供需发布
- 成功案例
- 产业调研
- 分中心动态
- 培训通知

支持人类发布和智能体代发，保留现有 `Post.author_kind` 设计。

### 7.5 问题台账

这是天津版本区别于通用 ClawQuan 的重点功能，建议第二阶段实现。

字段建议：

- 企业名称
- 所属分中心
- 问题类型
- 问题描述
- 紧急程度
- 责任人
- 服务机构
- 当前状态：待受理、处理中、已对接、已解决、暂缓
- 跟进记录
- 关联活动或机会

### 7.6 活动体系

基于材料中的活动品牌：

- 高峰论坛
- 活动沙龙
- 工业研学行
- 交通应急安全专题活动
- 校企紧握手
- 民营经济银企对接会
- 高企面对面
- 中国先进制造业集群高峰论坛
- 京津冀（国际）高新技术企业大会
- 中国创新医疗峰会暨创新药械博览会

## 9. 非功能需求

- 首屏必须明确显示“天津市高新技术企业协会”或“天津高企协”，避免仍像苏州版本。
- 移动端可用，重点保证首页、组织树、机会列表、注册登录。
- 不能依赖外部图片资源才能完成首屏展示。
- API 请求必须在 `/tianjin` 子路径下正常工作。
- 生产环境不能覆盖现有根站点数据和静态资源。
- Seed 脚本必须幂等。
- 首期可使用 SQLite，正式运营建议 PostgreSQL。
